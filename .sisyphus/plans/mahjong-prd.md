## 1. 概述

### 1.1 產品願景

打造一款流暢、直覺且具現代感的單人麻將遊戲，讓玩家能在瀏覽器中隨時隨地享受經典台灣 16 張麻將的樂趣。

### 1.2 產品定位

- **產品類型**：網頁遊戲 (Web Game)
- **目標平台**：Desktop / Mobile 瀏覽器
- **核心玩法**：單人對戰 AI (三個電腦玩家)
- **技術棧**：HTML5 + Vanilla JavaScript + CSS3 (無框架依賴)

### 1.3 目標用戶

- 麻將愛好者，想在電腦/手機上快速打發時間
- 喜歡簡約現代 UI 風格的玩家
- 不需要複雜註冊/登入的輕度遊戲需求

## 2. 功能範圍

### 2.1 MVP 版本 (核心優先)

| 功能 | 說明 | 優先級 |
|------|------|--------|
| 遊戲初始化 | 洗牌、砌牌、發牌、確定莊家 | P0 |
| 摸牌 | 每次輪到玩家時自動摸牌 | P0 |
| 打牌 | 玩家從手牌中選擇一張打出 | P0 |
| 吃牌 | 吃上家打的牌 (可選擇吃或不吃) | P0 |
| 碰牌 | 碰任一家打的牌 (可選擇碰或不碰) | P0 |
| 槓牌 | 槓任一家打的牌或摸到槓牌 | P0 |
| 胡牌 | 湊成胡牌牌型 (自摸/放槓) | P0 |
| 計分 | 根據台灣麻將計分規則計算輸贏 | P0 |
| AI 對手 | 三個電腦玩家自動決策 (簡單難度) | P0 |
| 重新開始 | 遊戲結束後可重新開始新局 | P1 |
| 悔牌功能 | 可選擇是否開啟 (遊戲設定) | P2 |

### 2.2 不包含在本版本 (Scope Out)

- 線上多人對戰 (需要後端伺服器)
- 複雜 AI 策略 (進階難度)
- 登入/註冊/排行榜
- 音樂與音效
- 複雜動畫效果
- 儲值/虛擬貨幣

## 3. 使用者流程

```
[開啟網頁] → [載入畫面] → [遊戲主畫面] → [遊戲進行中]
                                                     ↓
                              ┌────────────────────────┘
                              ↓
                    [胡牌結算] → [顯示結果] → [重新開始] / [離開]
```

### 3.1 主要流程說明

1. **開啟網頁**：載入遊戲資源 (麻將牌圖片、樣式)
2. **遊戲初始化**：電腦隨機發牌，決定莊家
3. **遊戲進行**：
   - 莊家先摸牌 → 打牌
   - 依序輪到 East → South → West → North
   - 玩家可吃/碰/槓/胡 (優先順序：胡 > 碰 > 吃)
   - AI 自動決策
4. **胡牌結算**：
   - 顯示胡牌牌型
   - 計算台數與分數
   - 顯示哪家胡牌 / 流局

## 5. 遊戲規則 (台灣麻將 16張)

### 5.1 基本規則

- **牌數**：144 张 (筒子 9x4、條子 9x4、萬子 9x4 + 字牌 8x4 + 花牌 8)
- **手牌**：開局每人 16 张，莊家 17 张
- **胡牌門檻**：3 台 (含) 以上
- **連莊**：莊家胡牌或流局，莊家繼續

### 5.2 吃碰槓規則

| 動作 | 條件 | 優先權 |
|------|------|--------|
| 胡牌 | 湊成 4 組 + 1 對子 | 最高 (無法放棄) |
| 碰 | 任一家打出的牌，手中有兩張相同 | 可放棄 |
| 吃 | 上家打出的牌，能湊成順子 | 可放棄 |
| 槓 | 手中有 3 張相同 + 摸到第 4 張 | 可放棄 |

### 5.3 計分規則

| 台數計算 | 台數 |
|----------|------|
| 門前清 (不吃碰槓) | +1 台 |
| 自摸 | +1 台 |
| 莊家 | +1 台 |
| 連莊 | 每連 +1 台 |
| 東南西北 (任一圈) | +1 台 |
| 中發白 (任一組) | +1 台 |
| 三元牌刻子 (刻子本身) | +2 台 |
| 花牌 | 每花 +1 台 |

### 5.4 流局

- 牌池剩餘 16 张以下 (海底撈月門檻)
- 四家同時流局

## 6. AI 行為規格

### 6.1 決策優先級

```
1. 如果能胡牌 → 必須胡牌
2. 如果能槓牌 → 評估是否槓 (保留好牌)
3. 如果能碰牌 → 評估碰牌風險 (是否放槓)
4. 如果能吃牌 → 吃牌後是否能聽牌
5. 否則 → 打出生肖/字牌 → 打孤張 → 打靠張
```

### 6.2 簡單難度特徵

- 不會刻意做牌
- 防守意識較低
- 隨機選擇吃/碰 (50% 機率)
- 打牌邏輯：字牌 > 孤張 > 靠張

## 7. 技術架構

### 7.1 檔案結構

```
mahjong-game/
├── index.html          # 遊戲主頁面
├── css/
│   └── style.css       # 所有樣式
├── js/
│   ├── game.js         # 遊戲主邏輯
│   ├── player.js       # 玩家類別
│   ├── ai.js           # AI 決策邏輯
│   ├── tiles.js        # 牌組管理
│   └── scoring.js      # 計分系統
└── assets/
    └── tiles/          # 麻將牌圖片 (可選，可純 CSS)
```

### 7.2 核心模組

| 模組 | 職責 |
|------|------|
| `Game` | 遊戲狀態管理、流程控制、事件派發 |
| `Player` | 手牌管理、吃碰槓胡判斷 |
| `AI` | 電腦玩家決策邏輯 |
| `Tiles` | 牌堆生成、洗牌、補牌 |
| `Scoring` | 台數計算、分數結算 |

### 7.3 資料結構

```javascript
// 牌型表示
{
  type: 'wan' | 'tiao' | 'tong' | 'zi' | 'hua',

}

// 手牌
[牌, 牌, 牌, ...]

// 副露 (吃碰槓)
[
  { type: 'pong', tiles: [牌, 牌], from: 'east' },
  { type: 'chow', tiles: [牌, 牌, 牌], from: 'south' },
  { type: 'kong', tiles: [牌, 牌, 牌, 牌], from: 'self' }

```
