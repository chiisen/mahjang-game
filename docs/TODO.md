## 🚀 優化項目清單

### Phase 1: 架構/工程化 ✅ 優先

- [ ] **引入建置工具 (Vite)**
  - 原因: 實現 tree-shaking、程式碼壓縮、依賴管理
  - 收益: 載入速度提升、程式碼品質提升

- [ ] **重構為 ES6 Modules**
  - 原因: 消除全域變數氾濫、明確依賴關係
  - 具體:
    - 將 `TileType`, `TileValue`, `ZI_NAMES`, `HUA_NAMES` 從全域改為 export
    - 將 `GameState`, `PlayerDirection`, `Logger` 改為 export
    - 使用 `import` 替代 script 標籤順序載入

### Phase 2: 程式碼品質 🟡 中期

- [x] **封裝模組 (IIFE 過渡方案)**
  - 如果暫不引入 bundler，使用 IIFE 減少全域污染

- [x] **修復 canChow 演算法**
  - 位置: `js/player.js:64-80`
  - 問題: 現有邏輯無法正確判斷順子組合
  - 修復: 實作完整的順子搜尋演算法

- [x] **增加程式碼註解**
  - 為每個 class 和重要函數添加 JSDoc

### Phase 3: AI 邏輯 🟡 中期

- [x] **移除隨機決策**
  - 位置: `js/ai.js:17, 23, 29`
  - 問題: `Math.random()` 導致不合理行為
  - 改進: 根據手牌價值、聽牌策略、場上資訊做決策

- [x] **實作基本牌譜策略**
  - 優先打孤張字牌
  - 保留潛在順子
  - 避免打出生張

### Phase 4: 計分系統 🟡 中期

- [ ] **實作完整番型**
  - [ ] 天胡 / 地胡 / 人胡
  - [ ] 海底撈月 / 河底撈魚
  - [ ] 大三元 / 大四喜
  - [ ] 黑白貓 / 紅中發財
  - [ ] 清一色 / 混一色
  - [ ] 對對胡 / 平胡

### Phase 5: 使用者體驗 🟢 後期

- [ ] **優化動畫時間**
  - 移除硬編碼 `setTimeout(..., 500)` 
  - 使用 CSS transition 替代

- [x] **增加 Loading 狀態**
  - 電腦思考時顯示旋轉動畫

- [ ] **改善互動回饋**
  - 牌疊 hover 效果優化
  - 選取狀態視覺強化

- [ ] **行動裝置支援**
  - 觸控事件處理

  - 可加入：背景音樂開關、動畫開關、遊戲速度

### Phase 6: 測試覆蓋 🟢 後期
